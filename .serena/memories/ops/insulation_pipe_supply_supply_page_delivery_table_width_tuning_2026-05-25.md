# 2026-05-25 insulation_pipe_supply_2026 供给页发货记录表恢复原结构并优化列宽

## 背景
用户明确指出：问题不在于是否需要做信息分组，而在于原表格里部分字段列宽太窄，内容显示不下，文本被挤压成串行换行。

## 调整结论
- 已撤回上一轮把发货记录表改成“分组块式表格”的做法。
- 当前恢复为原有列式表格结构。
- 优化方向改为：`colgroup + 单元格换行规则`。

## 变更文件
- `frontend/src/projects/insulation_pipe_supply_2026/pages/SupplyManagementView.vue`
- `configs/progress.md`
- `frontend/README.md`
- `backend/README.md`

## 具体实现
- 表头与数据单元恢复为原始 14 列：订单号、车次号、车牌号、供给主体、换热站、型号、三列数量、发货时间、在途时长、状态、备注、操作。
- 为表格增加专用 `colgroup`，按字段特性重新分配宽度：
  - 放宽 `订单号 / 车次号 / 车牌号 / 型号 / 发货时间 / 状态 / 备注 / 操作`
  - 数量列维持较窄固定宽度
- 调整换行规则：
  - 编号列、数量列、时间列使用 `nowrap`
  - 名称列、备注列允许 `word-break` 自然换行
- 保留操作列纵向按钮栈，只作为空间收纳，不改变表格信息结构。

## 结果
- 页面回到用户预期的标准台账表格形式。
- 主要问题从“结构改造”切回“列宽与排版调优”。
- 后端接口、字段、状态机、数量口径均无变化。

## 备注
- 本轮未做自动构建验证，原因仍是本地统一执行器启动失败，当前以静态代码核对为主。
